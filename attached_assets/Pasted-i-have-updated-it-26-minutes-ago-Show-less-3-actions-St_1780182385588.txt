The Idea
Right now, events search is generic — it looks at EIZ, ZACCI etc. and hopes for the best. The better model is:

"What events are the companies relevant to MY career involved in?"

A civil engineering student should see events where Bauer Zambia, Avic International, or Roads Development Agency are sponsoring, speaking, or exhibiting. A finance student should see events where Zambia National Commercial Bank, Madison Finance, or Deloitte are hosting workshops.

Proposed Architecture
Step 1 — Profile → Relevant Companies table

When a user saves/updates their profile, a background process runs and writes to a new table:

user_relevant_companies
─────────────────────────────────────────────
user_id         │ FK to users
company_name    │ e.g. "ZESCO Limited"
industry        │ e.g. "energy"
relevance_score │ 0–100
source          │ "local-db" | "ai-expanded"
updated_at      │ timestamp

This pulls from two sources:

The existing zambian_companies Supabase DB (already scored by profession)
An AI pass that adds well-known companies in their field not yet in the DB
Step 2 — Events search uses the company list

When the user opens the Networking tab, instead of just searching "career events Lusaka", the search becomes:

"ZESCO event workshop Zambia 2026"
"Barclays Bank Zambia career fair OR seminar 2026"
"First Quantum Minerals networking OR conference Zambia"
Run these in parallel for the top 8–10 companies. This surfaces events those companies host, sponsor, or appear at — which are the exact rooms you want to be in.

Step 3 — Trigger logic

When	Action
User saves profile (degree/industry changes)	Rebuild user_relevant_companies
User opens Networking tab	Check updated_at — if stale (>24h) or never built, rebuild
User taps "Refresh Events"	Optionally rebuild company list too
What I'll build
user_relevant_companies table in Supabase (migration)
Edge function handler build-company-list — takes profile, writes to table
Hook into profile save in profile.tsx — fires the builder after save
Update handleNetworkingEvents — reads the company list, generates targeted search queries per company
Progress step in the Networking tab — "Building your company list…" shown when rebuilding
